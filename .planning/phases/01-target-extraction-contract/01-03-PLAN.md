## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI `--project_dir` -> canonical repository | Caller-controlled filesystem identity must resolve to the runner's actual repository before any state change. |
| Canonical output candidate -> resolved `data/raw` | Existing directories/files can be symlinks; lexical path prefixes are not a safety boundary. |
| Pipeline exception -> integrity audit | Parse, extraction, plotting, or persistence failure must not bypass the post-run raw comparison. |
| Filesystem check -> subsequent write | A local concurrent actor could swap a validated path between resolution and write (TOCTOU residual). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-09 | Spoofing | `resolve_repository_root` | high | mitigate | Strictly resolve the CLI candidate and script-derived anchor, require equality plus repository markers, and reject before the first write. |
| T-01-10 | Tampering | `resolve_output_path` and all persistence destinations | high | mitigate | Resolve symlink components, require canonical-root containment and raw-tree exclusion, then revalidate after mkdir and immediately before JSON/NPZ/PNG writes. |
| T-01-11 | Tampering | validation-to-write TOCTOU window | medium | mitigate | Repeat path resolution at each concrete destination and verify raw bytes/metadata in `finally`; document the residual malicious concurrent swap as low likelihood for this single-user local pipeline. |
| T-01-12 | Repudiation | `run_pipeline` exception handling | high | mitigate | Put the second snapshot/comparison in `finally`; preserve clean-run exceptions, chain them behind integrity failures, and treat an incomplete audit as failure. |
