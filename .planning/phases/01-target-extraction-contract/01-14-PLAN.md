<objective>
Close UAT gap G-01-5 (status: failed, severity: major): boundary overlays remain fragmented and physically implausible even after the 01-05 continuity-tracking fix for G-01-2. `.planning/debug/boundary-fragmentation-post-continuity-fix.md` traces the root cause directly on the real 4-track data: `halfmax_edges`' tracked-candidate selection rule (`src/targets.py:161-171`) has NO maximum-distance gate and NO effective size/plausibility weighting -- length is only an unreachable exact-float-tie tiebreak -- so a 1-native-pixel noise spike and a 300-pixel true bead run are treated as equally legitimate candidates, decided purely by whichever midpoint happens to be marginally closer to `previous_center`. Combined with `extract_targets_from_arrays` unconditionally overwriting `previous_center` with whatever run was just selected, correct or wrong, ONE spurious column (a local noise dip briefly fragmenting or suppressing the true bead's above-threshold run) corrupts the anchor and triggers a MULTI-COLUMN self-reinforcing wrong lock -- a new failure mode the pre-01-05 independent-per-column algorithm did not have. Directly traced on Track 8 (x=24.9-26.5mm, an 8-column/1.4mm episode) and, far more severely, on Track 10 (a near-continuous narrow lock from x~68mm through beyond x~82mm). Measured wrong-narrow-pick rates: Track 8 5.7%, Track 10 7.1%, Track 14 34.3%, Track 21 43.1% of tracked selections. Reproduced exactly the reviewer's numbers: 22/65/63/34 finalized valid runs and 0.32-0.88mm max adjacent-column jumps for tracks 8/10/14/21.

This plan implements the diagnosis's two concretely-suggested, jointly-required mechanisms: (1) merge adjacent above-threshold sub-runs separated by only a few below-threshold samples before candidate enumeration, so a noise-fragmented true bead is not split into competing tiny candidates in the first place (fixes the TRIGGER, e.g. Track 10's x~67.1mm 3-way fragmentation); (2) gate tracked selection by candidate-length plausibility relative to the largest candidate present in that same column, so an implausibly-small candidate can never win purely on midpoint proximity (fixes the PROPAGATION, e.g. Track 8's 8-column wrong lock). A third suggested direction -- a full DP/Viterbi joint sequence tracker -- is explicitly named by the diagnosis as "not required to implement, if scope is too large"; given the project's 8-day runway and that mechanisms (1)+(2) directly and fully address both traced failure mechanisms, it is not implemented here (missing-information/context-cost tradeoff, not a scope reduction of a locked decision).

Purpose: TARGET-02 requires boundary overlays that are visually continuous and physically plausible, not merely gap-aware. A tracking rule with no plausibility floor at all is a materially weaker guarantee than 01-05's own stated intent and reintroduces a new, sometimes-worse failure mode than the original G-01-2 defect.
Output: `src/targets.py` with a `merge_adjacent_runs` helper and a plausibility-gated `halfmax_edges`/`extract_targets_from_arrays`; expanded `tests/test_targets.py` regressions reproducing the exact diagnosed Track 8 and Track 10 scenarios; Amendment A7; regenerated real 4-track artifacts with a quantitative fragmentation/jump-statistic before/after comparison and an honest ordering-outcome report.
