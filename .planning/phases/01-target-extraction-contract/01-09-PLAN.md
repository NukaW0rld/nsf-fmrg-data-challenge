|----------|-------------|
| filesystem name matching -> track identity | `find_track_file` maps a caller-supplied `track_id` onto a concrete file; a loose match silently rebinds a track's identity to another track's data. |
| filesystem read of `data/raw/` -> in-memory arrays | Raw height/thermal/SEM files are read-only inputs; nothing in this plan may write, delete, or modify under `data/raw/`. |
| symlinked SEM directory -> tile set | A symlink at `SEM_{track_id}` can redirect one track's tile listing to another track's images without any error. |
